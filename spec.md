# PlayMath 品牌网站技术规格文档

## 1. 项目概述

PlayMath是一个专注于数学内容创新的品牌，slogan是"像数学家一样玩"。通过寓教于乐、创新体验和亲子互动的方式，让数学内容好看和好玩。核心价值在于将抽象的数学概念转化为可视化、可互动的体验，激发儿童对数学的兴趣和探索欲。

### 核心数学概念
PlayMath的数学内容基于七个核心数学概念：
- **集合**：培养分类和分组能力
- **数感**：建立数字概念和数量关系
- **图形**：认识形状、对称和几何属性
- **测量**：理解长度、面积、体积等概念
- **空间关系**：发展空间思维和方向感
- **模式**：识别和创造规律
- **数据分析**：收集、整理和解读数据

### 问题解决
数学的刻板印象是抽象和枯燥的，导致儿童对数学产生畏惧心理。PlayMath通过展览、教具和玩具等多种形式，让数学好看好玩，帮助儿童建立数学思维和解决问题的能力。

### 目标受众
- **家长**：寻找优质数学教育资源，希望孩子在轻松愉快的环境中学习数学
- **学校**：寻求数学教育解决方案，丰富教学内容和方法
- **教育工作者**：需要教学辅助工具和资源
- **科技馆/商业空间**：寻求合作举办数学展览，提供教育性娱乐体验

### 品牌社会使命
PlayMath的社会使命是："让每一个孩子，都能触摸到数学之美"
- **公立学校合作**：对公立学校提供优惠以及免费的讲座或工作坊
- **社区合作**：长期与社区图书馆、公益教育机构合作，提供免费内容
- **教育公平**：致力于缩小教育资源差距，让更多孩子受益于优质数学教育

## 2. 用户流程与UI设计

### 页面结构
1. **首页**：品牌介绍、核心产品导航、展览亮点展示
2. **关于我们**：品牌故事、教育理念、团队介绍、社会使命
3. **数学展**："爱丽丝漫游数学王国"展览亮点、展区内容、核心展项
4. **产品中心**：数学教具和益智玩具展示
5. **常见问题**：FAQ常见问题解答
6. **联系我们**：联系方式、合作咨询、留言表单

### 交互流程
1. **用户进入网站**：通过首页了解品牌核心价值和主要产品
2. **浏览数学展**：查看"爱丽丝漫游数学王国"展览详情、核心展区介绍、预约参观
3. **探索产品**：浏览数学教具和益智玩具，了解产品特点和购买方式
4. **联系咨询**：通过联系方式或留言表单与品牌取得联系

### 设计风格
- **主色调**：黄色和橙色为主色，蓝色为辅助色，黄色和橙色代表活力和创造力，蓝色代表智慧和理性
- **视觉元素**：数学符号、几何图形、爱丽丝主题元素
- **布局风格**：高级简约风格，大面积留白，突出内容
- **背景设计**：大量使用照片作为背景底图，营造沉浸式视觉体验
- **图片资源**：使用 `picture/` 文件夹中的照片作为背景和配图
- **互动元素**：包含简单的数学互动游戏，增强用户体验，如：俄罗斯方块、2048等
- **响应式设计**：适配桌面端、平板和移动设备

## 3. 技术栈

### 前端技术
- **框架**：Vue 3
- **语言**：JavaScript
- **构建工具**：Vite + ViteSSG (静态站点生成)
- **样式预处理器**：SCSS
- **CSS框架**：Tailwind CSS
- **动画**：Vue Transition + CSS动画

### 外部依赖
- **图标库**：Font Awesome
- **托管服务**：Vercel / Netlify (自动部署)

### 技术方案优势
- **零服务器维护**：托管平台自动处理部署和运行
- **SEO友好**：ViteSSG生成静态HTML，搜索引擎友好
- **极速加载**：静态页面，首屏加载速度快
- **低成本**：托管服务通常有免费额度
- **易维护**：代码结构简单，内容更新通过Git提交

3. **样式系统**：配置SCSS和Tailwind CSS
4. **托管设置**：配置Vercel/Netlify部署

5. **首页开发**：实现品牌介绍、轮播图、导航等核心元素

## 10. 托管服务配置

### Vercel配置
1. **仓库连接**：将代码仓库连接到Vercel
2. **构建命令**：`npm run build`
3. **输出目录**：`dist`
4. **环境变量**：无需特殊配置

### Netlify配置
1. **仓库连接**：将代码仓库连接到Netlify
2. **构建命令**：`npm run build`
3. **发布目录**：`dist`
4. **环境变量**：无需特殊配置

## 11. 内容更新流程

1. **本地修改**：在本地修改内容文件
2. **代码提交**：`git add . && git commit -m "更新内容"`
3. **推送到远程**：`git push origin main`
4. **自动部署**：Vercel/Netlify自动构建并部署
5. **验证发布**：访问网站确认更新生效
